# Тест-кейсы — API тестирование микросервиса объявлений

**Host:** `https://qa-internship.avito.com`

## Эндпоинты

| # | Метод | Эндпоинт | Описание |
|---|-------|----------|----------|
| 1 | POST | `/api/1/item` | Создать объявление |
| 2 | GET | `/api/1/item/:id` | Получить объявления по его идентификатору |
| 3 | GET | `/api/1/:sellerID/item` | Получить все объявления по идентификатору продавца |
| 4 | GET | `/api/1/statistic/:id` | Получить статистику по айтем id |

---

## 1. POST /api/1/item — Создание объявления

### Позитивные сценарии

| ID | Название | Предусловия | Шаги | Ожидаемый результат |
|----|----------|-------------|------|---------------------|
| TC-1.1 | Создание с валидными данными | — | POST с sellerID=787878, name="Phone", price=79990, statistics={likes:1542, viewCount:8765, contacts:231} | 200, ответ содержит ID нового объявления |
| TC-1.2 | Созданное объявление доступно по ID | Создано объявление | GET /api/1/item/{id} | 200, поля name, price, sellerId совпадают с отправленными |
| TC-1.3 | Созданное объявление в списке продавца | Создано объявление | GET /api/1/{sellerID}/item | 200, объявление есть в массиве |
| TC-1.4 | Статистика совпадает с отправленной | Создано объявление | GET /api/1/statistic/{id} | 200, likes/viewCount/contacts совпадают |
| TC-1.5 | Граничное значение sellerID = 111111 | — | POST с sellerID=111111 | 200 |
| TC-1.6 | Граничное значение sellerID = 999999 | — | POST с sellerID=999999 | 200 |
| TC-1.7 | Несколько объявлений одного продавца | — | Создать 2 объявления с одним sellerID | 200 для обоих, оба в списке продавца |
